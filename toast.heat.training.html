toast.heat.training¶

Helper functions that perform many testing and visualization actions necessary when training a network

toast.heat.training.plot_training_loss(network, show_plot=False, save_plot=True, save_txt=False, save_path='./', output_folder_name='no_folder_name', path_log='./log.txt', close_plot=True)[source]¶

Uses a network object to generate the standard training/testing loss plot.

This is primarily a helper function to allow a single function call with a network object as the argument. For further plotting functionality description, the plotting specific actions are implemented by train_test_loss()

Parameters

  • network (toast network) – toast.heat generated network that has been trained so that it contains training history values to plot.

  • show_plot (bool, optional) – Whether to show the plot after generation

  • save_plot (bool, optional) – Whether to save the plot after generation. Note the filename will be determined based on attributes of the network’s path_network attribute

  • save_txt (bool, optional) – Whether to save a text file summary of the Train/Test/Accuracy values.

  • save_path (str, optional) – Path to the parent directory where output plots should be saved.

  • output_folder_name (str, optional) – Child directory to save the plots to.

  • path_log (str, optional) – Path to the log file for logging activity.

  • close_plot (bool, optional) – Whether to close the figure when finished. THis helps minimize memory usage.

Returns

No return value. Generates and displays/saves a plot

toast.heat.training.plot_distribution_weights(network, save_txt=True, show_plot=False, save_plot=True, save_path='./', output_folder_name='no_folder_name', path_log='./log.txt', close_plot=True)[source]¶

Uses a network object to generate distribution plot of the weights of each layer from the last training iteration

This will make separate plots for different layer types, so that similar layer types can be easily compared.

This is primarily a helper function to allow a single function call with a network object as the argument. For further plotting functionality description, the plotting specific actions are implemented by layer_distributions()

Note

Linear Layers contain a weights parameter module and a bias parameter module. (1 layer, 2 parameter modules)

Non-linearity Layers do not contain parameters as they only perform a fixed operation.

Parameters

  • network (toast network) – toast.heat generated network that has been trained so that it contains training history values to plot.

  • save_txt (bool, optional) – Whether to save a .txt summary of the final state of the varaibles for each layer

  • show_plot (bool, optional) – Whether to show the plot after generation

  • save_plot (bool, optional) – Whether to save the plot after generation. Note the filename will be determined based on attributes of the network’s path_network attribute
