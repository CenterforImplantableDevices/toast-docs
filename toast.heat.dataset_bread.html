toast.heat.dataset_bread¶

This module contains class definitions defining how data and datasets will be handled.

Data_Bread class acts as a wrapper class, and should be the object managing all of the data and segmentation required for a dataset to be used for ML applications.

Dataset_Bread class defines individual datasets that will be iterated over during training.

class toast.heat.dataset_bread.Data_Bread(path_data, path_sample_descriptions, label_key=None, needs_validation=True, one_hot_encode=False, data_files=None, data_type=<class 'numpy.int16'>)[source]¶

Bases: object

Data_Bread class acts as a wrapper class, managing all of the data and reladed sub-datasets required for a dataset to be used for ML applications.

Manages multiple Dataset_Bread instances automatically, making for a more cohesive user workflow so the user can manage data through a single class instance. Initialized with only a few arguments to setup the necessary datasets for training and testing ML models

class toast.heat.dataset_bread.Dataset_Bread(path_data, sample_descriptions, label_key, data_files=None, data_type=<class 'numpy.int16'>)[source]¶

Bases: torch.utils.data.dataset.Dataset

Dataset_Bread class defines individual datasets that will be iterated over during training. Inherits torch.utils.data.Dataset class for optimized implementation

Tip

User can override the preprocess() function below to add a data preprocessing step that will be executed every time a sample is loaded. See description below.

Attention

This class should not be directly used.

It is best to access this class as an attribute of the Data_Bread class for more streamlined dataset management.
