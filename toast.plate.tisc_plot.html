Separates TiSc data into a 2D array with each scale in one row, with each scale stretched to make an even length 2D array.

In this output each column index correlates wtih a unit of time.

  • data_dim (int, optional) – Dimension along which the TiSc organized data is organized.

  • scale_multiplier (int, optional) – Exponential base designating how fast the window size increases.

  • normalize (bool, optional) – Whether to independently normalize each scale to between 0 and 1.

Returns

2D arra where rows are each scale the the columns are each time index.

Return type

Numpy ndarray

toast.plate.tisc_plot.plot_inputWeights(network, num_channels_data, description='', show_plot=False, plot_dpi=100, plot_linewidth=0.5, save_plot=True, skip_plot=False, save_data=True, save_path='./', output_folder_name='no_folder_name', path_log='./log.txt', bkg_color='w', close_plot=True, use_agg=True)[source]¶

Uses a network object to separate the weights of the first tisc layer, sort into the proper channels, and plot the weights of the channel.

Function assumes that weights of each channel alternate within each filter window, matching the typical .wav file organization.

Function assumes the network as arranges following convention of nested sequential objets, as defined below:

  • Sequential[0] contains all TiSc-based network processing.

  • Sequential[0][*] contains one or many TiSc channels

  • Sequential[0][*][0] contains a TSC_input layer to initially process the input data

Parameters

  • network (toast network) – A toast.heat generated network that has been trained so that it contains training history values to plot.

  • num_channels_data (int) – Number of data channels to separate out.

  • description (str, optional) – A description of the data in the plot. This will be used as the filename.

  • show_plot (bool, optional) – Whether to show the plot after generation

  • plot_dpi (int, optional) – Resolution of the generated/saved plots.

  • plot_linewidth (float, optional) – Linewidth to use in the generated/saved plots.

  • save_plot (bool, optional) – Whether to save the plot after generation. Note the filename will be determined based on attributes of the network’s path_network attribute

  • skip_plot (bool, optional) – Whether to skip the plotting funcions. Use if only want to save data in pickle file.

  • save_data (bool, optional) – Whether to save the weight data in a pickle file after organization. Note the filename will be determined based on attributes of the network’s path_network attribute

  • save_path (str, optional) – Path to the parent directory where output plots should be saved.

  • output_folder_name (str, optional) – Child directory to save the plots to.

  • path_log (str, optional) – Path to the log file for logging activity.

  • bkg_color (str -or- tuple, optional) – Character or RBG Thruple representing the background color of the plot.

  • close_plot (bool, optional) – Whether to close the figure when finished. THis helps minimize memory usage.

  • use_agg (bool, optional) – Whether to use the Agg backend of matplotlib. Use when connecting to non-display machines/servers

Returns

No return value. Generates and displays/saves a plot. If plots are saved, a pickle file with the raw weight values will also be saved. File contians one list object with dimensions data[channel_tisc][scale][channel_eeg][raw_weights]
