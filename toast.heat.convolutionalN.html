Combines a Convolutional layer encoders with a mutlilayer perceptron ~toast.heat.perceptronN.PerceptronN decision/classifier component.

This is a convenience class to help easily initialize networks that follow a certain predefined organization. This organization includes some combination of convolutional layers, max-pooling layers, and avg-pooling layers as the first layers in a network, typically used as a “feature extraction” section. These layers can then be followed by fully connected layers typically used as a “calssifier” section. The organization/sequence of these layers is provided in the layer_desc parameter of this class initializaer. With this, you can define the properties of each layer according to the following ordered inputs for each layer.

Layer Type

layer_desc entry for that layer

Convolutional Layer

[ ‘conv’, [kernel_dimensions(1D,2D,or3D)], stride, number_repitions, dropout ]

Max-Pool Layer

[ ‘max’, [kernel_dimensions(1D,2D,or3D)], stride, dropout ]

Avg-Pool Layer

[ ‘avg’, [kernel_dimensions(1D,2D,or3D)], stride, dropout ]

Lineaer Layer

[ ‘linear’, [sequential layer sizes], dropout, keyword_args ]

Note that the linear layer definition is passed to toast_perceptron.define_sequential_linear_layers, check there for functionality

Example usage to replicate a network from “Efficient Epileptic Seizure Prediction Based on Deep Learning” -Daoud H, Bayoumi MA

network = ConvolutionalMlpN( [ 1, 23, 1280],
                          [ [  'conv', [2,3], 1, 32, False],
                            [   'max', [2,2], 2,      True],
                            [  'conv', [2,3], 1,  1, False],
                            [   'max', [2,2], 2,      True],
                            [  'conv', [2,3], 1,  1, False],
                            [   'max', [2,2], 2,      True],
                            [  'conv', [2,3], 1,  1, False],
                            ['linear', [None, 300, 100, 50, 20, 1], [True, True, True, True]]
                        ])

Important

Don’t forget to include a dimension for the number of input channels, likely a [1, …] or [3, …] dim before the data/image size

Parameters

  • input_shape (1D iterable) – 1D Iterable defining the input shape. Don’t forget to include a dimension for the number of input channels, likely a [1, …] or [3, …] dim before the data/image size

  • layer_desc (2D iterable) – 2D Iterable: sequential list contianing elemnt lists that describe each layers architecture. Specific indes values are defined in detail above.

  • dropout (1D iterable, optional) – 1D Iterable of Boolean Values: Sequentially decide if the correlating layer in layer_size iterable should be followed by a dropout layer. If value(s) don’t exist they will be set to False.

  • dropout_perc (float, optional) – The percentage dropout to implement in the dropotu layers.

  • nonlinearity (pyTorch Module, optional) – A pyTorch Module that representes the desired nonlinearity class for the entire network.

  • nonlinearity_final (pyTorch Module, optional) – A pyTorch Module that representes the desired nonlinearity class for the final output node.

  • path_save_progress (str, optional) – Path to the parent directory where progress and tracking data should be saved.

  • description (str, optional) – A description describing the contents/structure of the network for easy reference.

  • comment (str, optional) – A comment about what this network is being used to do/test/investigate for later identification of tests run.

Returns

An instance of the ConvolutionalN class representing a deep neural network.

Initialization function to create an instance of an Convolutional Encoder Decoder Network

This is a convenience class to help easily initialize networks that follow a certain predefined organization. This organization includes some combination of convolutional layers, max-pooling layers, and avg-pooling layers as the first layers in a network, typically used as a “encoding” section. These layers are then be followed by inverse layers of the same dimensions that map the encoded representation back to the original image in the “decoding” section. The organization/sequence of the encoding layers is provided in the layer_desc parameter of this class initializer. The inverse decoding layers are automatically generated. Using the the layer_desc parameter, you can define the properties of each layer according to the following ordered inputs for each layer.

Layer Type

layer_desc entry for that layer

Convolutional Layer

[ ‘conv’, [kernel_dimensions(1D,2D,or3D)], stride, number_repitions, dropout, batch_norm ]

Max-Pool Layer

[ ‘max’, [kernel_dimensions(1D,2D,or3D)], stride, dropout, batch_norm ]

Avg-Pool Layer

[ ‘avg’, [kernel_dimensions(1D,2D,or3D)], stride, dropout, batch_norm ]

Intended to replicate the TiSc architecture, but using CNN layers with time-invariant convolution.

Implementation and usage should exaclty match TiscMlpN() to allow for side-by-side comparison.

If length_input is set to None, then the largest window size is assumed to be the length of the input.

Parameters

  • layer_desc (2D iterable) – 2D Iterable: sequential list contianing elemnt lists that describe each layers architecture. Specific indes values are defined in detail above.

  • stride (int, optional) – Step size of each window. A each scale’s stride will be set to min(stride, window_length) to avoid skipping samples.

  • num_tisc_channels (int, optional) – How many repetitions or independent channels of Time Scale layers to apply to the data.

  • length_input (int, optional) – Size of the data that will be input into the network. Necessary if the largest TiSc window size does not cover the entire data length, to aid in dimension calculation.

  • dropout (1D iterable, optional) – 1D Iterable of Boolean Values: Sequentially decide if the correlating layer in layer_desc iterable should be followed by a dropout layer. If value(s) don’t exist they will be set to False.

  • dropout_perc (float, optional) – The percentage dropout to implement in the dropotu layers.

  • nonlinearity_tisc (torch activation, optional) – pytorch.nn activation fuction to use within the TiSc operation. Pass a reference to the class, not an instance of the class (don’t use parenthesis).

  • nonlinearity (torch activation, optional) – pytorch.nn activation fuction to use. Pass a reference to the class, not an instance of the class (don’t use parenthesis).

  • nonlinearity_final (torch activation, optional) – pytorch.nn activation fuction to use after the final layer. Pass a reference to the class, not an instance of the class (don’t use parenthesis).

  • tisc_scale_multiplier (int, optional) – Exponential base designating how fast the window size increases.

  • path_save_progress (str, optional) – Path to the parent directory where progress and tracking data should be saved.

  • description (str, optional) – A description describing the contents/structure of the network for easy reference.

  • comment (str, optional) – A comment about what this network is being used to do/test/investigate for later identification of tests run.

Returns

An instance of the ConvolutionalN class representing a deep neural network.

toast.heat.convolutionalN.define_sequential_conv_layers(input_shape, layer_desc, get_decode=False, dropout=[], dropout_perc=0.2, nonlinearity=<class 'torch.nn.modules.activation.ReLU'>, batchnorm=[])[source]¶

Streamlined to generate sequential object of containting standard layers in an order commonly seen in CNN architectures.

It is expected for most users to use the more comprehensive ConvolutionalN class when defining convolutional networks, however this function is spearated out for integration with other class definitions to maintain consistency within the toast framework.

Parameters

  • input_shape (1D iterable) – Expected dimensions of the input dat

  • layer_desc (1D iterable) – Sequential list of the description of each layer, starting with the input layer and ending with the output layer.

  • get_decode (bool, optional) – Whether to also assemble mirrored decoding layers

  • dropout (1D iterable, optional) – Iterable of Boolean Values: Sequentially decide if the correlating layer in layer_size iterable should be followed by a dropout layer. If value(s) don’t exist they will be set to False.

  • dropout_perc (float, optional) – Probability of an element to be zeroed.

  • nonlinearity (torch activation, optional) – pytorch.nn activation fuction to use. Pass a reference to the class, not an instance of the class (don’t use parenthesis).

  • batchnorm (1D iterable, optional) – Sequential list of whether or not to include batch normalization layers.

Returns

torch.nn.Sequential object containting the requested convolutional layers.
