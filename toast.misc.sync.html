toast.misc.sync¶

Contains class and function definitions that will synchronize output files in real time onto the CID synology drive.

toast.misc.sync._sync(target, destination, sleep_time)[source]¶

Target function run to watch a target path and periodically sync any changes to a destination path within the synology drive.

This syncronization is implemented using rsync with arguments rltvze.

Note

All sync commands are logged in a hidden .rsync-out file for later reference.

Parameters

  • target (str) – Folder path to sync (recursively, i.e. the entire folder tree) to the destination

  • destination (str) – Folder path within the Synology > ML Datasets to sync the target with.

  • sleep_time (int) – Seconds to wait in between sync updates.

Returns

No return value. Files are syncronized from target to destination until an exception is raised.

toast.misc.sync.start_sync(target_path, destination_path='', refresh=10)[source]¶

Streamlined function to start a continuously running thread to manage the real-time syncronization of files from a target path to a destination folder synology drive.

Destination path will be placed within the Synology > ML Datasets location.

This syncronization is implemented using rsync with arguments rltvze.

Parameters

  • target_path (str) – File path to monitor and syncronize files (recursively, i.e. the entire folder tree)

  • destination_path (str, optional) – File path within Synology > ML Datasets to place the syncronized files. Default is to placed the syncronized folder directly into the ML Datasets location

  • refresh (int, optional) – Seconds to wait between synchronizing changes.

Returns

Process object generated from the python multiprocessing.Process class.

toast.misc.sync.sync_now(target_path, destination_path='')[source]¶

Convenience function used to immediately syncronize a target path/file into the synology drive. A message will also be printed to stdout to inform the user of the syncronization action.

Destination path will be placed within the Synology > ML Datasets location.

This syncronization is implemented using rsync with arguments rltvze.

Parameters

  • target_path (str) – File path to syncronize (recursively, i.e. the entire folder tree if a folder is passed)

  • destination_path (str, optional) – File path within Synology > ML Datasets to place the syncronized files. Default is to placed the syncronized folder directly into the ML Datasets location

Returns

No return value. The target_path is synchronized with the synology drive.
